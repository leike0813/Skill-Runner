## 为什么要做

当前 `/ui` 管理界面只支持：

- 查看已安装 skill 列表
- 上传并安装 skill 包

还缺少面向单个 skill 的“包内容浏览”能力。用户无法在 UI 中检查 skill 包结构与关键文件内容，排障和审查效率较低。

另外，虽然系统已实现 Basic Auth，但需要进一步强化“可验证性与可观测性”，确保用户在本地和容器环境都能明确判断鉴权是否生效。

## 变更目标

1. 新增 skill 详情页面，支持浏览包结构（只读）。
2. 支持在页面中预览 skill 文件内容（只读，不可修改）。
3. 对二进制文件与大文件采取安全降级策略。
4. 增强 UI Basic Auth 的可观测性与测试覆盖，避免“误判未生效”。

## 变更范围

- 新增 `/ui/skills/{skill_id}` 与文件预览相关路由
- 新增 skill 浏览页面模板与局部组件
- 新增路径安全与文件预览限制策略
- 增强 UI 鉴权相关日志与测试
- 更新 API/UI 文档说明

## 非目标

- 不提供在线编辑能力（只读浏览）
- 不扩展到非 skill 目录的文件系统浏览
- 不在本次引入细粒度角色权限系统（沿用 Basic Auth）
