## 为什么要做

当前 Skill-Runner 已具备完整的技能查询与技能包安装 API，但缺少可直接操作的管理界面。  
对于运维和业务用户来说，仅通过 API 调用安装/查询技能的使用门槛较高。

我们需要新增一个 Web 管理界面，满足以下核心需求：

1. 可视化查看当前已安装技能及用途说明
2. 通过网页上传 Skill 包并触发安装
3. 在网页中查看安装状态
4. 提供基础鉴权，并支持容器环境变量初始化

## 变更内容

- 新增 `/ui` 管理界面（基于 FastAPI + Jinja2 + HTMX）
- 新增 UI 所需的页面路由与模板资源
- 在 UI 中接入现有技能查询与技能包安装能力
- 增加基础鉴权能力，并支持通过环境变量控制启用与初始化账号

## 影响范围

- 新增管理页面与静态资源（不影响现有 `/v1` API 契约）
- 新增/扩展配置项：UI 基础鉴权开关与凭据
- 更新容器化文档与 API 文档中的管理界面章节

## 非目标

- 不在本次变更中引入复杂 RBAC 或多用户体系
- 不在本次变更中替换现有 API 设计
- 不在本次变更中新增 Skill 编辑器，仅提供查看与安装入口
