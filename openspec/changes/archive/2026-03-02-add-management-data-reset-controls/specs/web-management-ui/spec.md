# web-management-ui Specification

## ADDED Requirements

### Requirement: 管理 UI MUST 提供高危数据重置入口并显著标识风险
系统 MUST 在管理 UI 中提供高危数据重置入口，并以明显视觉样式与文案提示其破坏性。

#### Scenario: 页面展示危险操作区
- **WHEN** 用户访问管理首页
- **THEN** 页面展示数据重置按钮与危险提示文案
- **AND** 按钮与容器具备显著风险标识，避免与常规操作混淆

### Requirement: 管理 UI MUST 通过弹窗与手动输入确认后才可执行重置
系统 MUST 在用户触发高危操作时弹出确认窗口，并要求手动输入指定确认文本后才允许提交执行请求。

#### Scenario: 未完成确认输入
- **WHEN** 用户打开确认弹窗但未输入正确确认文本
- **THEN** 执行按钮保持不可用或提交被阻断
- **AND** 不发起后端重置请求

#### Scenario: 输入正确确认文本
- **WHEN** 用户在弹窗中输入正确确认文本并提交
- **THEN** UI 调用管理 API 重置接口
- **AND** 请求中携带用户选择的可选清理开关

### Requirement: 管理 UI MUST 反馈重置结果并提示影响范围
系统 MUST 在操作完成后给出成功/失败反馈，并显示关键统计与影响说明。

#### Scenario: 重置执行成功
- **WHEN** 后端返回成功结果
- **THEN** UI 显示已删除/缺失/重建统计
- **AND** 提示用户相关运行数据与本地会话数据已被清理

#### Scenario: 重置执行失败
- **WHEN** 后端返回错误
- **THEN** UI 显示失败原因
- **AND** 保持当前页面可继续操作
